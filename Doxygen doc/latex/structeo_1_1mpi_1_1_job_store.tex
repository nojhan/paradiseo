\doxysubsection{Detailed Description}
\subsubsection*{template$<$typename Job\+Data$>$\newline
struct eo\+::mpi\+::\+Job\+Store$<$ Job\+Data $>$}

Contains all the required data and the functors to launch a job. 

Splitting the functors and data from the job in itself allows to use the same functors and data for multiples instances of the same job. You define your store once and can use it a lot of times during your program. If the store was included in the job, you\textquotesingle{}d have to give again all the functors and all the datas to each invokation of the job.

\mbox{\hyperlink{classeo_1_1mpi_1_1_job}{Job}} store contains the 4 functors (pointers, so as to be able to wrap them ; references couldn\textquotesingle{}t have permitted that) described above and the Job\+Data used by all these functors. It contains also helpers to easily wrap the functors, getters and setters on all of them.

The user has to implement data(), which is the getter for retrieving Job\+Data. We don\textquotesingle{}t have any idea of who owns the data, moreover it is impossible to initialize it in this generic \mbox{\hyperlink{structeo_1_1mpi_1_1_job_store}{Job\+Store}}, as we don\textquotesingle{}t know its form. As a matter of fact, the user has to define this in the \mbox{\hyperlink{structeo_1_1mpi_1_1_job_store}{Job\+Store}} subclasses.
